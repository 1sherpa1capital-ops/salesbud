# Email Threading with Resend

## Overview

To track email conversations with Resend and send properly threaded responses, you need to implement email threading using standard email headers (`Message-ID`, `In-Reply-To`, and `References`). This allows email clients to display messages as part of a conversation thread.

## How Email Threading Works

Email threading relies on three key headers:

1. **Message-ID**: A unique identifier for each email (generated by the sender)
2. **In-Reply-To**: The Message-ID of the email being replied to
3. **References**: A list of all Message-IDs in the conversation thread

## Implementation Strategy

### 1. Storing Conversation Data

You need a database to track conversations. Here's a suggested schema:

```sql
CREATE TABLE email_conversations (
    id UUID PRIMARY KEY,
    external_id VARCHAR(255) UNIQUE, -- Your internal reference (e.g., client_id + purpose)
    subject VARCHAR(500),
    recipient_email VARCHAR(255),
    recipient_name VARCHAR(255),
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE email_messages (
    id UUID PRIMARY KEY,
    conversation_id UUID REFERENCES email_conversations(id),
    resend_message_id VARCHAR(255), -- Resend's message ID from API response
    message_id VARCHAR(255), -- The Message-ID header value you generated
    in_reply_to VARCHAR(255), -- Message-ID of parent email
    subject VARCHAR(500),
    direction VARCHAR(10) CHECK (direction IN ('outbound', 'inbound')),
    sent_at TIMESTAMP DEFAULT NOW(),
    content TEXT
);
```

### 2. Sending the First Email (Starting a Thread)

When sending the initial email:

```javascript
import { Resend } from 'resend';
import { v4 as uuidv4 } from 'uuid';

const resend = new Resend(process.env.RESEND_API_KEY);

async function sendInitialEmail({ to, subject, html, conversationRef }) {
  // Generate a unique Message-ID
  const messageId = `<${uuidv4()}@syntolabs.xyz>`;

  // Send via Resend with custom headers
  const { data, error } = await resend.emails.send({
    from: 'team@syntolabs.xyz',
    to: to,
    subject: subject,
    html: html,
    headers: {
      'Message-ID': messageId,
      'X-Conversation-Ref': conversationRef // Custom header for your tracking
    }
  });

  if (error) throw error;

  // Store in database
  await db.query(
    `INSERT INTO email_messages (resend_message_id, message_id, subject, direction)
     VALUES ($1, $2, $3, 'outbound')`,
    [data.id, messageId, subject]
  );

  return { resendId: data.id, messageId };
}
```

### 3. Receiving Replies (Webhook Setup)

Configure a webhook to receive reply notifications:

```javascript
// Express webhook handler
app.post('/webhooks/resend', async (req, res) => {
  const { type, data } = req.body;

  if (type === 'email.received') {
    const {
      from,
      to,
      subject,
      text,
      html,
      headers
    } = data;

    // Extract threading headers
    const inReplyTo = headers['in-reply-to'];
    const references = headers['references'];
    const messageId = headers['message-id'];

    // Find the parent conversation
    let conversationId = null;

    if (inReplyTo) {
      // Look up by the message being replied to
      const parentMsg = await db.query(
        'SELECT conversation_id FROM email_messages WHERE message_id = $1',
        [inReplyTo]
      );

      if (parentMsg.rows.length > 0) {
        conversationId = parentMsg.rows[0].conversation_id;
      }
    }

    // Store the incoming message
    await db.query(
      `INSERT INTO email_messages
       (conversation_id, message_id, in_reply_to, subject, direction, content)
       VALUES ($1, $2, $3, $4, 'inbound', $5)`,
      [conversationId, messageId, inReplyTo, subject, text]
    );

    // Update conversation timestamp
    if (conversationId) {
      await db.query(
        'UPDATE email_conversations SET updated_at = NOW() WHERE id = $1',
        [conversationId]
      );
    }
  }

  res.status(200).send('OK');
});
```

### 4. Sending a Threaded Reply

When responding to a client reply:

```javascript
async function sendReply({ conversationId, html, text }) {
  // Get conversation history
  const conversation = await db.query(
    `SELECT * FROM email_conversations WHERE id = $1`,
    [conversationId]
  );

  const messages = await db.query(
    `SELECT * FROM email_messages
     WHERE conversation_id = $1
     ORDER BY sent_at ASC`,
    [conversationId]
  );

  const lastInbound = messages.rows
    .reverse()
    .find(m => m.direction === 'inbound');

  const lastOutbound = messages.rows
    .find(m => m.direction === 'outbound');

  // Build References header (all previous Message-IDs)
  const references = messages.rows
    .map(m => m.message_id)
    .join(' ');

  // Generate new Message-ID
  const newMessageId = `<${uuidv4()}@syntolabs.xyz>`;

  // Send reply with threading headers
  const { data, error } = await resend.emails.send({
    from: 'team@syntolabs.xyz',
    to: conversation.rows[0].recipient_email,
    subject: `Re: ${conversation.rows[0].subject}`,
    html: html,
    text: text,
    headers: {
      'Message-ID': newMessageId,
      'In-Reply-To': lastInbound?.message_id || lastOutbound?.message_id,
      'References': references
    }
  });

  if (error) throw error;

  // Store in database
  await db.query(
    `INSERT INTO email_messages
     (conversation_id, resend_message_id, message_id, in_reply_to, subject, direction)
     VALUES ($1, $2, $3, $4, $5, 'outbound')`,
    [conversationId, data.id, newMessageId, lastInbound?.message_id, `Re: ${conversation.rows[0].subject}`]
  );

  return data;
}
```

### 5. Looking Up Conversations

```javascript
async function getConversationHistory(conversationId) {
  const result = await db.query(
    `SELECT
       m.*,
       c.recipient_email,
       c.recipient_name
     FROM email_messages m
     JOIN email_conversations c ON m.conversation_id = c.id
     WHERE m.conversation_id = $1
     ORDER BY m.sent_at ASC`,
    [conversationId]
  );

  return result.rows.map(msg => ({
    id: msg.id,
    direction: msg.direction,
    from: msg.direction === 'outbound'
      ? 'team@syntolabs.xyz'
      : msg.recipient_email,
    subject: msg.subject,
    content: msg.content,
    sentAt: msg.sent_at,
    messageId: msg.message_id
  }));
}

// Or find by recipient email
async function findConversationsByEmail(email) {
  const result = await db.query(
    `SELECT * FROM email_conversations
     WHERE recipient_email = $1
     ORDER BY updated_at DESC`,
    [email]
  );
  return result.rows;
}
```

## Important Considerations

### Resend Limitations

1. **Inbound Email**: Resend primarily focuses on transactional email sending. For receiving replies, you may need:
   - A separate email service (like AWS SES, SendGrid Inbound Parse, or Mailgun)
   - Or use Resend's webhook capabilities if they support inbound email

2. **Webhook Verification**: Always verify webhook signatures:

```javascript
import crypto from 'crypto';

function verifyWebhookSignature(payload, signature, secret) {
  const expected = crypto
    .createHmac('sha256', secret)
    .update(payload)
    .digest('hex');

  return crypto.timingSafeEqual(
    Buffer.from(signature),
    Buffer.from(expected)
  );
}
```

### Alternative: Use a Dedicated Email API

For full conversation tracking, consider services like:
- **Nylas** - Full email sync and threading
- **Front** - Shared inbox with API
- **SendGrid** - Inbound parse webhook
- **Mailgun** - Routes and storage

### Simpler Approach: Subject-Based Threading

If full Message-ID threading is too complex, use subject-based threading:

```javascript
function normalizeSubject(subject) {
  // Remove Re: and Fwd: prefixes
  return subject
    .replace(/^(Re:|Fwd:)\s*/gi, '')
    .trim();
}

// Store normalized subject as conversation key
const conversationKey = normalizeSubject(subject);
```

This is less reliable but easier to implement.

## Summary

To track email conversations with Resend:

1. **Generate Message-IDs** for every email you send
2. **Store messages** in a database with threading metadata
3. **Set up webhooks** to receive replies
4. **Use In-Reply-To and References headers** when responding
5. **Query your database** to retrieve conversation history

The key is maintaining the relationship between emails through the Message-ID headers, which enables both your system and email clients to display proper conversation threads.
